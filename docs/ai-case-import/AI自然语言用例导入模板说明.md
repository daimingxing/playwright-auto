# AI 自然语言用例导入模板说明

## 使用场景

本模板用于批量整理自然语言测试用例。新用户默认使用两表模板：`用例清单` 和 `步骤明细`。填写后将 Excel 文件上传到平台，由平台生成可编辑的草稿用例。

每条用例必须有目标页面 URL。平台会从该地址进入页面，因此不要只写菜单名称、模块名称或业务入口说明。

旧三表模板仍作为兼容输入保留，可继续读取历史文件中的 `测试数据` 工作表；新文件不再要求填写 `测试数据`，输入值和检查期望值直接写在 `步骤明细` 的 `输入/期望值` 列。

## 工作表说明

新版模板包含两张业务工作表，可保留一张 `填写说明` 工作表作为 Excel 内提示。

| 工作表 | 用途 |
|---|---|
| 用例清单 | 一行填写一条用例的基本信息。 |
| 步骤明细 | 一行填写一个操作或检查步骤，按步骤序号排列。 |
| 填写说明 | 提供填写提示，不参与导入解析。 |

## 用例清单字段

| 字段 | 填写规则 |
|---|---|
| 用例编号 | 必填，同一文件内唯一，例如 TC001。 |
| 用例名称 | 必填，使用简短业务名称，例如“新增用户”。 |
| 目标页面URL | 必填，填写用例开始操作的页面地址。 |
| 前置条件 | 可选，填写登录状态、账号权限、已有数据等条件。 |
| 预期结果 | 可选，填写整条用例最终业务预期。 |
| 备注 | 可选，填写补充说明。 |

预期结果建议写成自然语言，例如“页面提示保存成功，用户列表中显示新建用户”。不要写成平台内部处理方式。

## 步骤明细字段

操作过程写在 `步骤明细`，不要写成一整段。每个动作拆成独立步骤，步骤序号从 1 开始连续递增。

| 字段 | 填写规则 |
|---|---|
| 用例编号 | 必填，必须对应用例清单中的用例编号。 |
| 步骤序号 | 必填，同一用例内按 1、2、3 递增。 |
| 动作类型 | 必填，使用中文下拉值，例如 `点击(click)`、`填写(fill)`、`检查文本(assertText)`。 |
| 目标类型 | 必填，使用中文下拉值，例如 `按钮(button)`、`输入框(input)`、`文本(text)`。 |
| 目标名称 | 必填，填写页面上的业务对象名称，例如“新增按钮”“用户名称输入框”。 |
| 输入/期望值 | 可选。输入步骤填写输入值，检查步骤填写期望值。 |
| 匹配方式 | 可选，检查文本、输入值、地址、标题时填写，例如 `包含(contains)`、`等于(equals)`、`正则(regex)`。 |
| 备注 | 可选，填写弹窗、跳转、新窗口等补充信息。 |

动作类型、目标类型和匹配方式会以中文展示给测试人员，括号内英文仅用于平台识别。不要只填写无法识别的自定义动作。

保存、提交、删除等步骤可以写入模板，用于生成待确认的草稿用例；页面地图采集阶段不会执行这些可能改变数据的危险动作，只会记录类似“已跳过危险动作：保存”的提示。

## 常见示例

新增用户用例：

| 表 | 推荐写法 |
|---|---|
| 用例清单 | TC001，新增用户，目标页面 URL 填 `/users`，预期结果写“页面提示保存成功，用户列表中显示新建用户”。 |
| 步骤明细 | 1 `点击(click)` `按钮(button)` `新增`；2 `填写(fill)` `输入框(input)` `用户名称`，输入/期望值为 `测试用户001`；3 `点击(click)` `按钮(button)` `保存`；4 `检查文本(assertText)` `文本(text)` `保存成功提示`，输入/期望值为 `保存成功`，匹配方式为 `包含(contains)`。 |

查询用户用例：

| 表 | 推荐写法 |
|---|---|
| 用例清单 | TC002，查询用户，目标页面 URL 填 `/users`，预期结果写“用户列表只显示符合查询条件的用户”。 |
| 步骤明细 | 1 `填写(fill)` `输入框(input)` `用户名称查询条件`，输入/期望值为 `测试用户001`；2 `点击(click)` `按钮(button)` `查询`；3 `检查文本(assertText)` `表格(table)` `用户列表`，输入/期望值为 `测试用户001`，匹配方式为 `包含(contains)`。 |

## 不推荐写法

| 不推荐写法 | 原因 |
|---|---|
| 把所有操作写在用例清单的备注中。 | 平台无法稳定识别步骤顺序。 |
| 只写“进入用户管理”。 | 缺少目标页面 URL。 |
| 动作类型写“提交(submit)”。 | `submit` 不是当前平台支持的动作枚举。 |
| 目标名称只写“按钮”。 | 缺少页面上的业务对象名称，AI 难以定位。 |
| 输入值仍放到旧 `测试数据` 工作表。 | 新版两表模板不会要求测试数据表，新文件应直接写 `输入/期望值`。 |
| 匹配方式写“模糊匹配”。 | 当前支持 `包含(contains)`、`等于(equals)`、`正则(regex)`。 |
| 预期结果写成内部检查方式。 | 预期结果应表达业务现象。 |
