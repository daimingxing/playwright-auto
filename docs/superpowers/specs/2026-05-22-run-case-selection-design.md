# 运行中心用例选择设计

## 背景

当前运行中心执行测试时，会运行项目下全部可用用例。新增需求是在运行中心选择本次要执行的部分用例，并提供全选、全不选的便捷操作。全不选时不能触发运行测试。

## 目标

- 运行中心支持勾选本次要运行的测试用例。
- 默认进入运行中心时选中全部可用用例，保持现有使用习惯。
- 支持全选和全不选。
- 没有选中用例时禁用运行测试按钮。
- 后端只运行前端传入的用例，并校验用例是否存在。
- 保持现有测试报告、登录态、环境、运行模式和并发设置流程不变。

## 非目标

- 不在用例管理页面增加长期启用或停用状态。
- 不修改用例文件结构。
- 不新增标签、分组、搜索或高级筛选。
- 不改变 Playwright 报告生成路径和报告列表展示逻辑。

### 运行设置区

运行设置区保留现有字段：

- 运行环境
- 运行模式
- 并发数
- 登录态提示
- 打开浏览器登录
- 保存登录态
- 运行测试

运行按钮文案随选择状态变化：

- 已选中用例时显示 `运行已选 N 条`。
- 未选中用例时显示 `请选择用例`，并禁用按钮。
- 运行中沿用加载状态。

运行按钮禁用条件：

- 没有保存登录态。
- 没有选中任何用例。
- 当前正在运行。

### 测试用例区

测试用例区放在运行设置右侧，负责本次运行范围选择。

顶部展示：

- `已选择 N / M 条`
- `全选`
- `全不选`

列表字段：

- 勾选框
- 用例名称
- 起始路径
- 更新时间

用例为空时展示空状态，并禁用运行按钮。

### 响应式布局

宽屏或中等宽度下使用左右分栏：

- 左侧：运行设置。
- 右侧：测试用例。

窄屏下改为上下布局：

- 运行设置。
- 测试用例。
- 测试报告。

## 前端设计

### 状态

`RunCenter.vue` 新增状态：

- `cases`：当前项目可用用例列表。
- `selectedCaseKeys`：本次选中的用例 key。

加载运行中心时同时加载：

- 项目信息。
- 运行配置。
- 用例列表。
- 登录态状态。
- 报告列表。

默认选择全部用例：

- `cases` 加载完成后，将所有 `case.key` 写入 `selectedCaseKeys`。
- 如果后续刷新用例列表，需要保留仍然存在的已选 key，并自动移除不存在的 key。

### 操作

新增函数：

- `selectAllCases()`：选中全部当前可用用例。
- `clearCases()`：清空本次选择。
- `getSelectedCount()`：计算已选数量。
- `canRun()`：统一判断运行按钮是否可用。

运行测试时，调用 `runProject()` 并传入：

```ts
{
  envKey: selectedEnv.value,
  mode: runMode.value,
  workers: workers.value,
  caseKeys: selectedCaseKeys.value
}
```

## 后端设计

### 共享类型

`shared/types.ts` 的 `RunInput` 增加：

```ts
caseKeys?: string[];
```

### 运行服务

`runProject()` 接收 `caseKeys` 后，只运行选中的用例。

处理规则：

- `caseKeys` 未传入时，保持兼容，运行全部可用用例。
- `caseKeys` 是空数组时，抛出 `请选择至少一条测试用例`。
- `caseKeys` 包含不存在的用例 key 时，抛出 `选择的测试用例不存在或已被删除`。
- 过滤后没有可运行用例时，抛出 `当前项目没有可运行用例` 或更具体的选择错误。

`getProjectRunFiles()` 调整为支持可选筛选：

```ts
getProjectRunFiles(projectKey: string, caseKeys?: string[])
```

该函数内部仍然使用当前项目已有的 `listCases()`，再按 `caseKeys` 过滤，最终生成 Playwright 文件过滤参数。

## 异常处理

前端阻止明显无效操作：

- 全不选时禁用运行按钮。
- 用例为空时禁用运行按钮。

后端做最终校验：

- 防止绕过前端直接请求空数组。
- 防止传入已删除或不存在的用例 key。
- 保持错误信息为测试人员可理解的中文。

## 测试设计

### 前端测试

补充或调整运行中心相关测试：

- 加载运行中心后默认全选用例。
- 点击全不选后运行按钮禁用。
- 点击全选后恢复选择。
- 运行测试时请求体包含选中的 `caseKeys`。

### 后端测试

补充运行服务或运行接口测试：

- 未传 `caseKeys` 时仍运行全部用例。
- 传入部分 `caseKeys` 时只生成对应用例过滤参数。
- 传入空数组时报错。
- 传入不存在的用例 key 时报错。
