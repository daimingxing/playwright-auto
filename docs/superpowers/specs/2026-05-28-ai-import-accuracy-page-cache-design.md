# AI 用例导入正确率优化设计

## 目标

当前 AI 用例导入链路已经能完成“模板导入 -> AI 推理 -> 生成草稿用例”，但生成正确率偏低。主要原因不是单个模型调用失败，而是模板输入过于自由、页面上下文只包含初始静态可见元素、同一页面的多条用例重复独立处理。

本设计目标是把 AI 从“猜测用例步骤”调整为“基于结构化模板和页面地图生成草稿”。平台继续负责确定性解析、页面信息采集、缓存、基础检查和保存；AI 只负责把结构化输入与页面地图合成为可编辑草稿。

## 已确认范围

- 做模板简化，测试人员直接选择动作类型。
- 动作类型、目标类型、匹配方式使用“中文(英文)”形式展示，例如“点击(click)”。
- 取消独立“测试数据”工作表，把输入值和期望值直接写在步骤行。
- 增加页面信息缓存，避免同一目标页面每次导入都重新采集。
- 页面缓存默认长期有效，提供手动刷新、删除和查看。
- 增加受控动态探索，缓存菜单展开、弹窗打开、页签切换、下拉展开后的页面信息。
- 同一环境、同一目标 URL 的多条用例共用页面缓存，并尽量合并到同一次 AI 输入中生成。

## 暂不处理

- 不做 AI 自由 agent，让 AI 自己决定点击哪里。
- 不做“基础检查失败后自动让 AI 修复一次”的修复循环。
- 不在导入阶段点击保存、提交、删除、审批、支付、发送等高风险动作。
- 不自动覆盖已有草稿用例。
- 不引入云端队列或分布式缓存。
- 不把完整 HTML 直接交给模型。

## 总体流程

```text
上传 Excel
-> 解析新版两表模板
-> 按环境和目标 URL 分组导入项
-> 查找页面地图缓存
-> 无缓存时生成初始页面地图
-> 需要时执行受控动态探索并更新页面地图
-> 同 URL 分组调用 AI 生成多条草稿
-> 校验 AI 输出结构
-> 执行现有基础检查
-> 用户在导入预览页确认并保存草稿
```

## 模板设计

### 工作表

新版模板采用两张工作表：

```text
用例清单
步骤明细
```

取消“测试数据”工作表。输入值、查询条件、检查期望值直接写在步骤明细的“输入/期望值”列中。

### 用例清单

| 列名 | 是否必填 | 说明 |
|---|---|---|
| 用例编号 | 必填 | 同一文件内唯一，例如 `TC001`。 |
| 用例名称 | 必填 | 保存草稿时作为用例名称来源。 |
| 目标页面URL | 必填 | 用例开始页面，平台按该地址查找或生成页面地图。 |
| 前置条件 | 可选 | 登录态、权限、已有数据等前置说明。 |
| 预期结果 | 可选 | 用例整体预期，可辅助 AI 生成检查步骤。 |
| 备注 | 可选 | 补充说明。 |

### 步骤明细

| 列名 | 是否必填 | 说明 |
|---|---|---|
| 用例编号 | 必填 | 关联用例清单中的用例编号。 |
| 步骤序号 | 必填 | 同一用例内从 1 连续递增。 |
| 动作类型 | 必填 | 使用下拉选项，例如“点击(click)”。 |
| 目标类型 | 按动作必填 | 使用下拉选项，例如“按钮(button)”。 |
| 目标名称 | 按动作必填 | 页面上测试人员可见的对象名称，例如“新增”“用户名称”。 |
| 输入/期望值 | 按动作可选 | 输入动作填写输入值，检查动作填写期望值。 |
| 匹配方式 | 按检查动作可选 | 检查类动作使用，例如“包含文本(contains)”。 |
| 备注 | 可选 | 补充说明，例如“点击后打开新增用户弹窗”。 |

### 动作类型

动作类型展示中文，括号内保留内部英文值。导入解析时只保存英文枚举。

```text
打开页面(goto)
点击(click)
右键点击(rightClick)
双击(doubleClick)
悬停(hover)
填写(fill)
选择(select)
等待(wait)
检查文本(assertText)
检查可见(assertVisible)
检查输入值(assertValue)
检查地址(assertUrl)
检查标题(assertTitle)
```

### 目标类型

目标类型用于帮助平台和 AI 缩小定位范围。

```text
页面(page)
按钮(button)
输入框(input)
下拉框(select)
链接(link)
菜单(menu)
页签(tab)
弹窗(dialog)
文本(text)
表格(table)
树节点(tree)
日期控件(date)
区域(region)
```

### 匹配方式

匹配方式只服务检查类动作。普通点击、填写、选择步骤可以为空。

```text
包含文本(contains)
完全等于(equals)
正则匹配(regex)
元素可见(visible)
输入值等于(valueEquals)
地址包含(urlContains)
地址等于(urlEquals)
标题包含(titleContains)
标题等于(titleEquals)
```

## 页面地图缓存

### 页面地图定义

页面地图是目标页面在某个环境、登录态和视口下的可复用页面信息。它不等同于完整 HTML，也不只是初始静态页面摘要。

页面地图包含：

- 页面基本信息：地址、标题、标题文本。
- 初始可见元素：按钮、输入框、下拉框、链接、菜单、表格等。
- 受控探索后的状态：菜单展开、弹窗打开、页签切换、下拉展开后的元素。
- 每个元素的候选定位器和唯一性检查结果。
- 采集告警：未登录、页面文本过少、元素不唯一、探索动作失败等。

### 缓存键

页面地图缓存键至少包含：

```text
projectKey
envKey
targetUrl
authHash
viewport
```

`authHash` 来自登录态文件的更新时间或内容摘要。不同账号权限可能看到不同页面，所以不能只按 URL 缓存。

### 存储位置

建议存储在项目数据目录下：

```text
data/projects/<projectKey>/page-maps/<mapId>/
  map.json
  snapshots/
    <snapshotId>.json
```

`map.json` 保存页面地图摘要和索引，`snapshots` 保存具体状态快照。

### 数据结构草案

```ts
interface PageMap {
  mapId: string;
  projectKey: string;
  envKey: string;
  targetUrl: string;
  finalUrl: string;
  authHash: string;
  viewport: {
    width: number;
    height: number;
  };
  status: 'ready' | 'stale' | 'failed';
  states: PageState[];
  warnings: string[];
  createdAt: string;
  updatedAt: string;
}

interface PageState {
  stateId: string;
  name: string;
  sourceAction?: PageAction;
  page: {
    url: string;
    title: string;
    headings: string[];
  };
  elements: PageElements;
  warnings: string[];
}

type PageElements = PageContext['elements'];

interface PageAction {
  type: 'click' | 'hover' | 'select' | 'open';
  targetType: string;
  targetName: string;
  safe: boolean;
}
```

最终实现可以复用现有 `PageContext`，但需要在其外层增加状态和动作路径。

### 过期策略

页面地图默认长期有效，不做强制自动删除。

平台展示缓存年龄，并在以下情况标记为建议刷新：

- 缓存超过配置天数，默认 7 天。
- 登录态文件变化。
- 目标 URL 实际跳转地址变化。
- 页面地图采集失败。
- 基础检查或实测检查频繁出现元素找不到。

用户可在页面地图管理入口执行：

- 查看缓存。
- 刷新缓存。
- 删除缓存。

## 受控动态探索

### 探索原则

受控动态探索由平台决定可执行动作范围，不让 AI 自由操作浏览器。

允许探索的动作必须满足：

- 不会提交业务数据。
- 不会删除或修改业务记录。
- 只用于暴露更多页面结构。
- 可根据模板步骤或页面元素类型明确判断。

### 默认允许

```text
展开菜单
切换页签
打开弹窗
展开折叠面板
打开下拉框
打开日期控件
展开树节点
悬停显示浮层
```

### 默认禁止

```text
保存
提交
删除
移除
审批
确认
支付
发送
导入
导出
批量操作
启用
禁用
```

禁止动作只记录元素存在，不执行点击。

### 探索来源

探索动作优先来自导入模板中的前置安全步骤。例如多条用例都包含：

```text
点击(click) | 菜单(menu) | 系统管理
点击(click) | 菜单(menu) | 用户管理
点击(click) | 按钮(button) | 新增
```

平台可以在生成页面地图时执行这些安全路径，并把每一步后的页面快照存入页面地图。

### 探索上限

为了控制风险和耗时，建议设置上限：

- 每个目标 URL 最多探索 20 个安全动作。
- 每条动作路径最多 4 层。
- 每类元素最多保留 30 个候选。
- 单次页面地图生成超时默认 60 秒。

这些值应写入配置文件，便于后续调整。

## 同 URL 分组生成

### 分组键

导入任务中的用例按以下字段分组：

```text
projectKey
envKey
targetUrl
authHash
viewport
```

同组用例共用同一份页面地图。

### AI 输入结构

同组 AI 输入包含：

- 页面地图摘要。
- 本组所有用例的用例清单信息。
- 本组所有用例的结构化步骤。
- 输出规则。

输入示意：

```json
{
  "pageMap": {},
  "cases": [
    {
      "caseNo": "TC001",
      "caseName": "新增用户",
      "targetUrl": "/user/list",
      "steps": []
    }
  ]
}
```

### AI 输出结构

AI 必须按用例编号返回多条草稿：

```json
{
  "items": [
    {
      "caseNo": "TC001",
      "draft": {
        "name": "新增用户",
        "startPath": "/user/list",
        "steps": [],
        "confidence": "medium",
        "warnings": [],
        "missingInfo": []
      }
    }
  ]
}
```

如果某条用例无法生成，模型仍应返回该用例的失败原因，不能影响同组其他用例。

### 降级策略

如果同组输入过大或模型返回不可用：

- 先按用例数量拆分小批次。
- 再降级为单条用例生成。
- 降级时继续复用页面地图缓存。

## 页面地图管理入口

建议在项目详情页增加“页面地图”入口，或在 AI 导入页增加缓存状态区域。

最小可用展示：

```text
目标页面
环境
登录态
最近采集时间
缓存状态
已采集状态数量
操作：查看 / 刷新 / 删除
```

AI 导入预览页应显示当前导入项使用的页面地图：

```text
页面地图：/user/list，default，已采集 6 个状态，更新时间 2026-05-28 15:20
```

## 与现有实现的关系

当前页面上下文采集入口在 `server/src/services/ai/page-context.ts`。新设计应尽量复用已有元素读取函数，把单次 `PageContext` 扩展为可缓存的多状态 `PageMap`。

当前导入队列在 `server/src/services/import/import-worker.ts` 中逐条处理导入项。新设计需要改为先按页面分组，再按组查找或生成页面地图，最后按组调用 AI。

当前 Excel 解析在 `server/src/services/import/import-excel.ts` 中读取三张工作表。新设计需要兼容新版两表模板，并可考虑短期保留旧模板解析以降低迁移成本。

当前 AI 草稿生成在 `server/src/services/ai/ai-case-draft.ts` 中单条生成。新设计应新增分组生成入口，保留单条生成作为降级路径。

## 配置建议

在 `playwright-auto.config.json` 中增加页面地图配置：

```json
{
  "ai": {
    "pageMap": {
      "staleDays": 7,
      "maxActions": 20,
      "maxDepth": 4,
      "timeoutMs": 60000,
      "autoCreate": true
    }
  }
}
```

字段含义：

- `staleDays`：超过多少天提示建议刷新，不强制删除。
- `maxActions`：单个页面地图最多执行的安全探索动作数量。
- `maxDepth`：单条探索路径最多层数。
- `timeoutMs`：单次页面地图生成超时时间。
- `autoCreate`：导入时没有缓存是否自动创建页面地图。

## 安全策略

- 页面地图采集阶段只执行安全动作。
- 危险关键词命中的按钮或菜单默认不点击。
- 平台应在页面地图中记录“已跳过危险动作”，便于用户理解为什么某些动态结果没有采集。
- 对弹窗、toast、message 的采集只发生在安全动作之后。
- 保存、提交、删除等业务副作用只生成草稿步骤，不在导入采集阶段执行。

## 测试策略

### 单元测试

- 新模板动作类型中文英文值解析。
- 目标类型中文英文值解析。
- 匹配方式中文英文值解析。
- 新版两表模板解析。
- 缺少必填字段时返回中文错误。
- 步骤序号不连续时返回中文错误。
- 页面地图缓存键生成。
- 登录态变化导致缓存标记建议刷新。
- 危险动作识别。
- 同 URL 分组逻辑。

### 服务测试

- 上传新版模板后能生成导入任务。
- 同 URL 多条用例只生成或读取一次页面地图。
- 页面地图生成失败时导入项进入失败状态并保留错误信息。
- 同组 AI 输出部分失败时，不影响其他用例进入待确认。
- 输入过大时能降级拆批或单条生成。

### 前端测试

- AI 导入页面能展示页面地图缓存状态。
- 用户能刷新和删除页面地图。
- 导入预览能展示用例使用的页面地图信息。
- 新模板字段展示不出现内部英文枚举裸值。

## 验收标准

- 测试人员可以使用新版两表模板导入用例。
- 动作类型、目标类型、匹配方式均支持中文(英文)下拉值。
- 输入值和检查期望值可以直接写在步骤明细中，不再依赖测试数据工作表。
- 同一目标 URL 的多条用例可以共用页面地图缓存。
- 页面地图缓存可以查看、刷新、删除。
- 页面地图能记录初始页面和受控探索后的多个状态。
- 导入阶段不会点击危险动作。
- AI 生成草稿时能使用页面地图中的候选定位器。
- 同 URL 分组生成失败时有降级路径。
- 现有单条导入能力保留为兼容或降级能力。

## 实施建议

建议拆成四个开发阶段：

1. 模板简化：新版两表模板、解析、prompt、导入预览展示和文档更新。
2. 页面地图缓存：缓存存储、缓存键、查看、刷新、删除、初始静态页面地图。
3. 受控动态探索：安全动作识别、菜单/弹窗/页签/下拉探索、多状态页面地图。
4. 同 URL 分组生成：导入队列分组、分组 AI 输入输出、降级策略。

每个阶段都应补充对应单元测试和必要前端测试。第一阶段完成后即可替换当前模板，提高输入确定性；后续阶段逐步提升页面信息质量和批量生成一致性。
